## What stays generated

- `docs/reference/configurations.mdx` is generated from `plugin.configs.recommended` and `plugin.configs.strict`.
- `docs/reference/rules/index.mdx` and each rule page under `docs/reference/rules` are generated from rule metadata plus the first valid and invalid test cases.
- Re-run `pnpm run docs:generate` after changing rule descriptions, fixability, configs, or tests.

## What stays hand-written

- Installation, configuration, and scope guidance stay hand-written because they explain how to use the package in a real project.
- Generated reference pages stay narrow and factual so they can track the source without turning the docs into a README copy.

## Read the code first

The source package lives in `packages/eslint-plugin-gadget`. Rule descriptions come from each rule's `meta.docs.description`, while examples come from the existing rule tests. That keeps the docs aligned with the behavior you actually ship.
