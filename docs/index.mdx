An opinionated, community-built ESLint plugin that catches common mistakes in [Gadget](https://gadget.dev) action files. It ships 16 rules across three categories, auto-fixes where possible, and offers `recommended` and `strict` presets.

## What it checks

<CardGroup cols={3}>
  <Card title="Options and params" icon="gear">
    Validates that your `options` and `params` exports contain only values Gadget can serialize and process.
  </Card>
  <Card title="Runtime safety" icon="shield-check">
    Prevents empty handlers, missing types, and patterns that cause transaction timeouts or billing surprises.
  </Card>
  <Card title="Enqueue safety" icon="arrow-right-to-bracket">
    Enforces explicit retries and concurrency limits on `api.enqueue()` calls.
  </Card>
</CardGroup>

## Quick start

```bash
pnpm add -D @aurelienbbn/eslint-plugin-gadget eslint
```

```js title="eslint.config.js"
import gadget from "@aurelienbbn/eslint-plugin-gadget";

export default [gadget.configs.recommended];
```

Run `eslint .` and the plugin handles the rest - it only activates on Gadget action file paths.
