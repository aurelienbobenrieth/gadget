Require explicit retries option on api.enqueue calls to avoid unintended retry behavior.

| Property | Value |
| --- | --- |
| Type | `suggestion` |
| Fixable | No |
| Recommended | `warn` |
| Strict | `error` |
| Scope | All files (any file containing `api.enqueue()` calls) |
| Category | Enqueue safety |

## Reported messages

- **`missingRetries`**: Set api.enqueue retries explicitly. Gadget defaults to 6.

## Examples

### Incorrect

```ts
api.enqueue(api.publish, {})
```

### Correct

```ts
api.enqueue(api.publish, {}, { retries: 0 })
```
