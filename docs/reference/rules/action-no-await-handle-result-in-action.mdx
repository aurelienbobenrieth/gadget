Disallow awaiting handle.result() inside run or onSuccess exports in Gadget actions.

| Property | Value |
| --- | --- |
| Type | `suggestion` |
| Fixable | No |
| Recommended | `warn` |
| Strict | `error` |
| Scope | Gadget action files (global and model) |
| Category | Runtime safety |

## Reported messages

- **`inOnSuccess`**: Do not await handle.result() in onSuccess, it bills the full wait time.
- **`inRun`**: Do not await handle.result() in run, it causes GGT_TRANSACTION_TIMEOUT.

## Examples

### Incorrect

```ts
export const run = async () => {
  const handle = await api.enqueue(api.publish, {});
  await handle.result();
};
```

### Correct

```ts
export const onSuccess = async () => {
  await items.map(async (item) => {
    await handle.result();
  });
};
```
