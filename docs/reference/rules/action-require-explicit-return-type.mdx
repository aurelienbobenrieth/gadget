Require returnType to be explicitly set in the options export of Gadget actions.

| Property | Value |
| --- | --- |
| Type | `suggestion` |
| Fixable | Yes (auto-fix available) |
| Recommended | `warn` |
| Strict | `error` |
| Scope | Gadget action files (global and model) |
| Category | Options and params |

## Reported messages

- **`missingReturnType`**: Set returnType explicitly in options.

## Examples

<Tabs>
  <Tab title="Incorrect">

```ts
export const options: ActionOptions = { actionType: "create" };
```

  </Tab>
  <Tab title="Auto-fix output">

```ts
export const options: ActionOptions = { actionType: "create", returnType: false, };
```

  </Tab>
  <Tab title="Correct">

```ts
export const options: ActionOptions = { returnType: true };
```

  </Tab>
