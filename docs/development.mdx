## Preview locally

Run Mintlify from the `docs` directory:

<Steps>
  <Step title="Install the Mintlify CLI">```bash npm i -g mint ```</Step>
  <Step title="Start the preview server">```bash cd docs mint dev ```</Step>
</Steps>

The preview server watches your MDX and `docs.json` updates at `http://localhost:3000`.

## Regenerate the reference pages

```bash
pnpm run docs:generate
```

- The command rebuilds `@aurelienbbn/eslint-plugin-gadget` first.
- It regenerates `docs/reference/configurations.mdx`.
- It regenerates `docs/reference/rules/index.mdx` and each rule page under `docs/reference/rules`.

## Generated inputs

- Rule descriptions come from `meta.docs.description` in `packages/eslint-plugin-gadget/src/rules/**/rule.ts`.
- Config severities come from `plugin.configs.recommended` and `plugin.configs.strict`.
- Example snippets come from the first `invalid` and `valid` test cases in each `rule.test.ts` file.

## Review checklist

- Run `pnpm run docs:generate` after rule or config changes.
- Start `mint dev` in `docs` and verify the sidebar structure.
- Check a generated rule page after editing tests so the example snippets still read well.

## When to edit by hand

- Edit the hand-written guide pages when the install flow, usage recommendations, or maintenance workflow changes.
- Edit the generator when you want to expose more rule metadata in the reference.
